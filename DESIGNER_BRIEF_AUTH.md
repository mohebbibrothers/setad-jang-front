# 🎨 بریف طراحی — صفحات احراز هویت (ورود / ثبت‌نام)

> **پلتفرم:** بعثت مردم · **مخاطب:** گرافیست تیم
> **هدف:** طراحی UI کامل، حرفه‌ای و یکدست با هویت برند برای تمام مسیرهای ورود و ثبت‌نام
> **زمان تحویل پیشنهادی:** هم‌زمان با اتمام صفحه اصلی

---

## ۱) جمع‌بندی سریع از Backend (آنچه باید UI پشتیبانی کند)

اپ `apps/authentication` یک سیستم احراز هویت **حرفه‌ای، چندشناسه‌ای، با OTP و امنیت بالا** است. خلاصه قابلیت‌ها:

| فیچر | توضیح |
|---|---|
| **چندشناسه‌ای** | کاربر می‌تواند با **ایمیل** یا **شماره موبایل (E.164)** ثبت‌نام کند |
| **OTP** | کد ۵ رقمی · اعتبار ۵ دقیقه · حداکثر ۵ تلاش · cooldown ۶۰ ثانیه بین درخواست‌ها |
| **ورود ۲ مسیره** | با **رمز عبور** یا **OTP** (به انتخاب کاربر) |
| **بازیابی رمز** | از طریق OTP روی همان identifier (ایمیل یا تلفن) |
| **مدیریت شناسه‌ها** | افزودن identifier دوم، تغییر شناسه‌ی اصلی (نیاز به OTP) |
| **پروفایل** | نام، نام خانوادگی، کد ملی، تاریخ تولد، جنسیت، آواتار، استان، شهر، آدرس |
| **مدیریت نشست‌ها** | نمایش لیست device های فعال، revoke هر نشست به‌صورت تکی |
| **High-trust profile** | برای جوایز R4J، پروفایل کامل (کد ملی، تولد، جنسیت، آدرس) لازم است |
| **Anti-abuse** | honeypot، rate-limit، global IP guard، تشخیص ریسک (device جدید، IP جدید…) |

## ۲) صفحاتی که باید طراحی شوند

برای پوشش **همه‌ی flow های بک‌اند** این صفحات لازم است:

### 🔐 Auth flow اصلی (Public)
1. **ورود** (`/login`) — دو tab: «با رمز عبور» / «با کد یکبارمصرف»
2. **ثبت‌نام — مرحله ۱** (`/signup`) — وارد کردن identifier
3. **ثبت‌نام — مرحله ۲ تأیید OTP** (`/signup/verify`)
4. **ثبت‌نام — مرحله ۳ تکمیل پروفایل** (`/signup/complete`) — رمز + نام + نام خانوادگی
5. **بازیابی رمز — مرحله ۱** (`/forgot-password`)
6. **بازیابی رمز — مرحله ۲ تأیید + رمز جدید** (`/forgot-password/verify`)

### 👤 صفحات کاربر (Authenticated)
7. **پروفایل من** (`/profile`) — تب‌های اطلاعات شخصی، شناسه‌ها، نشست‌های فعال، تغییر رمز

## ۴) جزئیات هر صفحه

### 🟢 صفحه ورود (Login)

**عنوان:** «خوش آمدید — وارد حساب کاربری شوید»

**کامپوننت‌های لازم:**
- **Tab Switcher** (ورود با رمز / ورود با کد یکبارمصرف) — pill segmented
- **فیلد ورود identifier** — یک input هوشمند که هم ایمیل و هم شماره موبایل را قبول کند
  - placeholder: «ایمیل یا شماره موبایل»
  - باید بتواند **آیکن کوچک معتبر/نامعتبر** سمت چپ نمایش دهد
- **فیلد رمز عبور** (فقط در تب رمز) — با toggle نمایش/مخفی رمز (eye icon)
- **چک‌باکس «مرا به خاطر بسپار»**
- **لینک «رمز عبور را فراموش کرده‌اید؟»** (سمت چپ، کوچک‌تر)
- **دکمه CTA اصلی** — `bg-brand-500`, gradient اختیاری, h-48px, rounded-full
  - متن: «ورود» / «دریافت کد ورود» (بسته به tab)
- **سپراتور «یا»** (خط افقی با متن وسط)
- **لینک ثبت‌نام:** «حساب کاربری ندارید؟ ثبت‌نام کنید»

### 🟢 صفحه ثبت‌نام — مرحله ۱ (Identifier)

**عنوان:** «به جمع ما بپیوندید»
**زیرعنوان:** «با ایمیل یا شماره موبایل خود شروع کنید»

- **فیلد identifier** (همان input هوشمند بالا)
- **چک‌باکس «شرایط و قوانین را می‌پذیرم»** (با لینک به `/terms`)
- **دکمه CTA:** «ارسال کد تأیید» (mint)
- **لینک:** «حساب دارید؟ وارد شوید»

### 🟢 صفحه ثبت‌نام — مرحله ۲ (OTP Verify)

**عنوان:** «کد تأیید را وارد کنید»
**زیرعنوان:** «کد ۵ رقمی به ایمیل/شماره‌ی **{مقدار masked}** ارسال شد»
*مثال masked: `a***@gmail.com` یا `0912***1234`*

- **OTP Input** — ۵ خانه‌ی مجزا، هر کدام input تک رقمی، auto-jump به بعدی
  - استایل: کادر مربعی، border ۲px، radius 12px، font درشت (18-22px)، tabular-nums
  - active state: border brand-500 + glow ملایم
- **Timer countdown** — نمایش `«ارسال مجدد کد در 00:45»` با progress ring اختیاری
- **دکمه «ارسال مجدد»** — disabled وقتی timer فعال است، active بعد از پایان
- **دکمه CTA:** «تأیید و ادامه»
- **لینک:** «تغییر ایمیل/شماره» (back به مرحله ۱)
- **پیام خطا** — وقتی کد اشتباه باشد، با شمارش `«۲ تلاش از ۵»`

### 🟢 صفحه ثبت‌نام — مرحله ۳ (Profile)

**عنوان:** «اطلاعات حساب خود را تکمیل کنید»

- **نام** + **نام خانوادگی** (دو فیلد در یک ردیف)
- **رمز عبور** + **تکرار رمز عبور** (دو فیلد در یک ردیف، با toggle نمایش)
- **Password Strength Meter** — یک نوار افقی با ۴ بخش:
  - ضعیف (قرمز) / متوسط (نارنجی) / خوب (زرد) / عالی (سبز)
  - زیر آن متن کوتاه: «حداقل ۸ کاراکتر، شامل حروف بزرگ و کوچک، عدد و کاراکتر خاص»
- **دکمه CTA:** «تکمیل ثبت‌نام» (مین، با لودینگ state)

### 🟢 صفحات بازیابی رمز (Forgot Password)

دو صفحه مشابه ثبت‌نام (مرحله ۱ identifier + مرحله ۲ OTP و رمز جدید)

### 🟣 صفحه پروفایل کاربر (Authenticated)

Layout: sidebar چپ + content راست

**Sidebar — منوی پروفایل:**
- 👤 اطلاعات شخصی
- 📧 شناسه‌ها (مدیریت ایمیل و موبایل)
- 🔒 امنیت (تغییر رمز)
- 💻 نشست‌های فعال (devices)
- 🚪 خروج

**صفحه «اطلاعات شخصی»:**
- آواتار با دکمه آپلود (دایره ۱۲۰px، border با gradient brand)
- نام + نام خانوادگی
- کد ملی (با validation ۱۰ رقم)
- تاریخ تولد (date picker شمسی)
- جنسیت (radio: مرد / زن)
- استان + شهر (cascading dropdown)
- آدرس (textarea)
- بادج «پروفایل کامل» (سبز) یا «پروفایل ناقص» (نارنجی) بالای صفحه
  - اگر ناقص باشد، لیست فیلدهای باقی‌مانده نمایش داده شود

**صفحه «شناسه‌ها»:**
- نمایش لیست (ایمیل + موبایل)
- هر شناسه: badge «اصلی» (طلایی) یا «ثانویه»، badge «تأیید شده» (سبز چک)، دکمه «حذف» (قرمز کوچک)
- دکمه «افزودن شناسه جدید» (اگر هنوز یکی نباشد)
- دکمه «تنظیم به‌عنوان شناسه‌ی اصلی»

**صفحه «نشست‌های فعال»:**
- لیست device ها، هر کارت:
  - آیکن device (لپ‌تاپ / موبایل / تبلت — استخراج‌شده از user_agent)
  - نام device + browser
  - IP و موقعیت تقریبی
  - آخرین فعالیت (relative time)
  - badge «نشست فعلی» سبز
  - دکمه «پایان دادن نشست» قرمز (به جز نشست فعلی)
- دکمه بزرگ پایین: «خروج از همه‌ی دستگاه‌ها»

## ۶) جزئیاتی که گرافیست باید لحاظ کند

### ⚠️ State های بصری هر کامپوننت
هر input/button باید **۵ state** داشته باشد:
- `default`
- `hover` (دسکتاپ)
- `focus` (ring brand-500)
- `disabled` (opacity 0.5، cursor not-allowed)
- `error` (border قرمز + پیام خطا زیر)

### 📱 Responsive
- **mobile-first**: زیر 640px فرم تمام عرض، پنل برند به header کوتاه تبدیل
- **tablet 640-1024**: split 50/50
- **desktop 1024+**: split 60/40 (فرم 60، برند 40)

### ♿ دسترسی‌پذیری
- placeholder رنگ کافی برای کنتراست (WCAG AA)
- لیبل‌های واضح بالای فیلدها
- اعداد OTP با `inputmode="numeric"` و `autocomplete="one-time-code"` (برای auto-fill موبایل)

### ✨ Micro-interactions (ایده‌ها)
- وقتی OTP کامل وارد می‌شود → کمی pulse سبز روی همه ۵ خانه
- success state — یک checkmark بزرگ سبز + animated stroke
- loading state — spinner داخل دکمه + متن «در حال ارسال...»
- error shake — خطای رمز اشتباه با shake کوتاه افقی

## ۸) ایده‌های نوآورانه (اختیاری ولی high-impact)

این‌ها گرافیست را آزاد می‌گذارد تا با خلاقیت پیاده کند:

### 🎬 پنل برند جان‌دار
به‌جای تصویر استاتیک، یک **video loop کوتاه** از مدافعان (با gradient overlay) یا یک **پارالاکس** ساده از موج‌های دات‌دار.

### 🌗 Adaptive identifier hint
وقتی کاربر ایمیل تایپ می‌کند، آیکن کوچک ایمیل ظاهر شود؛ وقتی شماره تایپ می‌کند، آیکن گوشی — به محض شناسایی اولین ۲ کاراکتر.

### 🏆 Trust score بعد از signup
بعد از تکمیل ثبت‌نام، یک «Trust Score» دایره‌ای (مثل امتیاز اعتبار) نمایش داده شود که بسته به تکمیل فیلدها افزایش می‌یابد. این انگیزه‌ای برای تکمیل پروفایل است.

### 🌍 Smart location pre-fill
در فیلد «استان/شهر»، اگر دسترسی به GPS داده شد، پیشنهاد خودکار.

### 🎯 Onboarding ۳ مرحله‌ای پس از signup
- «به ما بگو چه می‌خواهی بکنی؟» — کارت‌های انتخابی: «حمایت مالی»، «ارائه‌ی کمک»، «یادگیری»، «گزارش‌گری»
- شخصی‌سازی dashboard بعدی بر اساس انتخاب
